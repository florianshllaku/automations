🎬 Project Overview

This project is a fully automated AI content repurposing pipeline. It monitors rundown.ai for new AI-industry news articles, sends them to the user for approval via Telegram, and — upon approval — autonomously generates a complete short-form social video (TikTok / Instagram Reels / YouTube Shorts format).

The pipeline uses Claude Opus 4.6 for research and scriptwriting, ElevenLabs for voiceover, fal.ai for vertical 9:16 image generation, AssemblyAI for word-level subtitle transcription, and ffmpeg for final video assembly — delivering every asset back to the user over Telegram.

Input
Where content comes from

rundown.ai AI news category — scraped with Playwright headless browser, up to 4 pages per run.

Approval
Human-in-the-loop

Each new article is sent to the user via Telegram with YES / NO inline buttons before any processing begins.

Output
What gets produced

MP4 video (1080×1920, h.264, AAC) + SRT subtitles + 8–12 generated JPEGs + MP3 voiceover — all delivered via Telegram.

Visual Style
Current active style

Algorithmic — high-contrast dark backgrounds, glowing grid lines, circuit-board aesthetics, bold sans-serif overlays. (3 other styles are defined but inactive.)

⚙️ End-to-End Pipeline Flow

Complete sequence from startup to final video delivery, triggered by running python scraper.py.

1
Scraping Phase
Playwright headless browser opens rundown.ai/articles?category=AI. Scrapes up to 4 pages of article cards. Stops at checkpoint (most recently seen article). Saves new articles to seen_articles.json.
scraper.py → main() → scrape_page()
2
User Approval Phase
For each new article, sends a Telegram message with YES/NO inline buttons (5s gap between messages). Waits for user responses. Pipeline proceeds in real-time as approvals arrive.
scraper.py + telegram_bot.py → send_approval_message() → wait_for_next()
3
Article Research
Playwright fetches full article text. Claude Opus 4.6 extracts: main topic, 5 key facts, hook angle, best quote. Hero image downloaded if available.
pipeline.py → research_article() → _call_claude()
4
Reel Script Generation
Claude Opus 4.6 converts research into a punchy 45–50s TikTok/Reels script with labeled segments (HOOK, CONTEXT, BUILD, TWIST, CTA), timestamps, on-screen text, and voiceover text.
pipeline.py → generate_reel_script()
5
Voiceover Generation
All segment voiceovers concatenated and sent to ElevenLabs (eleven_multilingual_v2). MP3 saved to audio_files/. Delivered to user via Telegram.
voice.py → generate_audio() + telegram_bot.py → send_audio()
6
Visual Prompt Design
Claude Opus 4.6 generates 8–12 detailed image prompts from the reel script, using the "algorithmic" style specification. Each frame includes bold embedded text overlays as part of the prompt.
pipeline.py → generate_visual_prompts()
7
Image Generation
All prompts submitted to fal.ai (non-blocking, 5s gap). Collected in order once complete. 9:16 vertical JPEGs downloaded locally. Auto-resized if >10MB. All images sent to Telegram.
image_gen.py → generate_images_for_visuals() + telegram_bot.py → send_generated_images()
8
Subtitle Generation
Audio uploaded to AssemblyAI. Transcription requested with word-level timestamps. Polled until complete. Words grouped into ~10-word chunks (2-line format) with 50ms gaps. SRT file saved and sent to Telegram.
video.py → generate_srt()
9
Video Assembly
Per-image durations calculated from voiceover word counts (clamped 6–10s, rescaled to audio length). ffmpeg concat demuxer created. Video filter chain: fps=24, scale 1080×1920, pad black, burn subtitles. Encoded as h.264 (crf=23, fast) + AAC 192k. Final MP4 saved to output_videos/.
video.py → generate_video()

🔒 Security Notes

⚠
API Keys in .env File The .env file contains live API keys for Anthropic, ElevenLabs, fal.ai, AssemblyAI, and Telegram. While this file is correctly listed in .gitignore, it is stored unencrypted on disk. Never commit it, share it, or expose it. Consider rotating keys if this project directory has ever been shared.
!
No .env.example Template There is no .env.example file documenting which environment variables are required. If you set up this project on a new machine, you need to know all 7 variable names by reading the source code. Consider adding a template with placeholder values.
!
ffmpeg / ffprobe Not in requirements.txt These are required system binaries but are not documented in any dependency file. A fresh setup will fail at the video assembly step without them installed and in PATH.
✓
.gitignore is Correct The .gitignore properly excludes .env, seen_articles.json, __pycache__/, and *.pyc. Sensitive files and state files will not be accidentally committed.
